# weatherapi
microservices for weather api
```
VARIABLE DE ENTORNO
PROFILE_ACTIVE=dev;

---------------------------------------
EL PROYECTO CUENTA CON DOS MICROSERVICIOS

1.
NOMBRE: LOGIN
DESCRIPCION: SERVICIO PARA OBTENER TOKEN JWT DE AUTORIZACION
METODO: GET
URL: http://localhost:9095/weather/login (CONFIGURACION LOCAL)
PARAMETROS:
user:test
password:pwd
RESPUESTA
{
    "username": "test",
    "token": "Bearer {TOKEN}"
}
 
UNA VEZ GENERADO EL TOKEN DE AUTORIZACIÃ“N, ESTE DEBE DE SER COLOCADO A NIVEL DE 
HEADERS EN LAS DEMAS PETICIONES

2.
NOMBRE: GET WEATHER INFO
DESCRIPCION: RETORNA LA INFORMACION CLIMATICA DEL DIA SIGUIENTE SEGUN LA CIUDAD INGRESADA
METODO: POST
URL: http://localhost:9095/weather/weathercity (CONFIGURACION LOCAL)
HEADERS:
Authorization: Bearer {TOKEN}
PARAMETROS:
city: {ciudad}
RESPUESTA



PARA MAYOR REFERENCIA, SE COMPARTE LA COLECCION DE POSTMAN EN LA CARPETA "postmanCollection"
DEL RESPECTIVO CONSUMO DE LOS SERVICIOS
```
